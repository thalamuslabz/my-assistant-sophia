# Engineering Gates

## Gate 0 — Planning Required
- Sprint plan approved and stored in `planning/SPRINT_PLAN_v1.md`
- Scope aligns with v1 local-only requirements
- Risks and dependencies documented

## Gate 1 — Implementation Start
- Architecture and module breakdown reviewed
- Test plan mapped to deliverables
- Evidence collection process ready

## Gate 2 — Feature Complete
- All acceptance criteria met per sprint
- No critical bugs
- UI checklist satisfied

## Gate 3 — Release Readiness
- All required tests executed
- Test results recorded in `/test-evidence/`
- SHA256 hashes recorded in `planning/TEST_EVIDENCE_INDEX.md`
- Release criteria met and documented
